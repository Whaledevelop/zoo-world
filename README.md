# Zoo World

Небольшая Unity-симуляция пищевой цепочки: лягушки и змеи появляются в мире, перемещаются по своим стратегиям, обходят препятствия и взаимодействуют при столкновениях.

## Архитектура

Проект разделён на модели, системы, представления и настройки:

- **Models** хранят состояние без привязки к сцене. `AnimalModel` описывает отдельное животное, а `AnimalsModel` содержит коллекцию, счётчики и реактивные события жизненного цикла.
- **Systems** реализуют игровую логику и работают через модели и сервисы. Системы не вызывают друг друга напрямую.
- **Views** связывают модели с объектами Unity: `AnimalView` синхронизирует физическое тело и сообщает о столкновениях, `CameraView` и `ZooUIView` предоставляют сериализованные компоненты сцены.
- **Settings** — `ScriptableObject`-конфигурации животных, движения, спавна, камеры, препятствий, пула и UI.
- **Services и registries** инкапсулируют запросы к физике, экранным границам и соответствие моделей их представлениям.

Зависимости собираются в `ZooLifetimeScope` через VContainer. `ZooEntryPoint` запускает и освобождает все реализации `IAsyncInitializable` и `IAsyncReleasable`. Для реактивного состояния и событий используется R3, для асинхронного жизненного цикла — UniTask.

Основной поток данных:

1. `SpawnSystem` создаёт `AnimalModel`, получает `AnimalView` из пула и регистрирует животное в `AnimalsModel`.
2. `MovementSystem` выбирает стратегию движения по типу животного и обновляет `Rigidbody` в `FixedUpdate`.
3. `AnimalView` передаёт столкновения в `AnimalsModel`.
4. `FoodChainSystem` определяет хищника и жертву, после чего модель публикует событие смерти.
5. `AnimalDespawnSystem` возвращает представление в пул, а `UISystem` обновляет счётчики и визуальные эффекты.

## Основные сущности

- `AnimalModel` / `IAnimalModel` — состояние животного: идентификатор, тип, группа, позиция, скорость и признак жизни.
- `AnimalsModel` / `IAnimalsModel` — единый источник состояния животных, статистики и событий столкновения, поедания, появления и удаления.
- `AnimalView` — сериализованные `Rigidbody`, `Collider` и направление модели; синхронизация физики с моделью.
- `AnimalSettings` — тип, группа, префаб и настройки движения животного.
- `AnimalType` — доступные виды: `Frog` и `Snake`.
- `AnimalGroup` — роль в пищевой цепочке: `Prey` или `Predator`.

## Основные скрипты

| Скрипт | Назначение |
|---|---|
| `ZooLifetimeScope` | Регистрирует модели, системы, сервисы, представления и настройки. |
| `ZooEntryPoint` | Управляет асинхронной инициализацией и освобождением систем. |
| `SpawnSystem` | Периодически создаёт животных в свободных точках мира с учётом лимита пула. |
| `MovementSystem` | Обновляет движение животных через стратегии. |
| `FrogJumpMovementStrategyAsset` | Управляет фазами ожидания, поворота, прыжка и приземления лягушки. |
| `SnakeLinearMovementStrategyAsset` | Управляет непрерывным движением, поворотами и обходом препятствий змеи. |
| `SnakeObstacleBounceSystem` | Меняет направление змеи после столкновения с препятствием. |
| `FoodChainSystem` | Обрабатывает столкновения добычи и хищников. |
| `AnimalDespawnSystem` | Удаляет погибшие модели и возвращает их представления в пул. |
| `BoundsSystem` | Удерживает животных внутри заданных границ мира. |
| `CameraSystem` | Применяет положение и параметры ортографической камеры. |
| `UISystem` | Обновляет статистику и показывает эффект поедания. |
| `ObstacleQueryService` | Выполняет проверки свободных позиций и путей через Unity Physics. |
| `AnimalViewsPool` | Переиспользует представления животных по их `AnimalSettings`. |

## Структура проекта

- `Assets/_game/Scripts` — игровой код по функциональным модулям.
- `Assets/_game/Scriptables` — конфигурации игры и стратегии движения.
- `Assets/_game/Prefabs` — представления животных и связанные объекты.
- `Assets/_game/Main.unity` — основная сцена.
- `Assets/_whaledevelop` — базовые абстракции жизненного цикла, систем, диспетчеров, пулов и UI.
